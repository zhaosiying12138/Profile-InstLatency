Status: preliminary scaffold.

patch. It records where calibrated evidence must land once Phase 7 and Phase 8
produce accepted profile fields.

## Non-Claims

- `ReadAdvance` is not generated by the draft because no consumer-specific
  evidence exists yet.
- Physical writeback latency is not an LLVM schedule field here and is not
  independently claimed.
- Any generated LLVM implementation must keep unknown fields conservative rather
  than inventing values for completeness.
